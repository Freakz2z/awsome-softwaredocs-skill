# 序列图模板 (Sequence Diagram)

## 模板说明

序列图（Sequence Diagram）用于描述对象之间按时间顺序交互的消息序列。

## 基本语法

```mermaid
sequenceDiagram
    participant A as 参与者/对象
    participant B as 参与者/对象

    A->>B: 同步消息
    B-->>A: 响应消息
    A->>+B: 异步消息(激活B)
    B->>-A: 消息结束
    A-xB: 销毁消息
    loop 循环描述
        B->>A: 循环内容
    end
    alt 条件1
        A->>B: 条件分支1
    else 条件2
        A->>C: 条件分支2
    end
    opt 可选
        A->>B: 可选内容
    end
```

## 消息类型说明

| 语法 | 消息类型 | 说明 |
|------|----------|------|
| `->>` | 同步消息 | 发送者等待响应 |
| `-->>` | 响应消息 | 返回值 |
| `->>+` | 激活+同步 | 激活目标对象 |
| `-->>-` | 结束激活 | 销毁对象 |
| `-x` | 异步消息 | 发送者不等待 |
| `loop` | 循环片段 | 循环执行 |
| `alt/else` | 选择片段 | 条件分支 |
| `opt` | 可选片段 | 可选执行 |

## 模板示例

### 1. 用户登录序列图

```mermaid
sequenceDiagram
    participant U as 用户
    participant W as Web前端
    participant A as 认证服务
    participant DB as 数据库

    U->>W: 输入用户名密码
    W->>A: 提交登录请求
    A->>DB: 查询用户信息
    DB-->>A: 返回用户数据
    A->>A: 验证密码
    A->>A: 生成JWT Token
    A-->>W: 返回Token和用户信息
    W-->>U: 登录成功，跳转首页
```

### 2. 订单创建序列图

```mermaid
sequenceDiagram
    participant C as 客户
    participant S as 购物车服务
    participant O as 订单服务
    participant P as 支付服务
    participant I as 库存服务
    participant M as 消息服务

    C->>S: 发起结算
    S->>S: 计算商品总价
    S-->>C: 显示订单确认页

    C->>S: 确认订单
    S->>O: 创建订单(待支付)
    O-->>S: 返回订单ID

    C->>P: 选择支付方式发起支付
    P->>P: 调用支付接口
    P-->>C: 支付中...

    P-->>O: 支付成功回调
    O->>O: 更新订单状态(已支付)
    O->>I: 扣减库存
    I-->>O: 库存已扣减

    O->>M: 发送订单通知消息
    M-->>C: 通知客户订单已确认

    O-->>P: 返回支付确认
    P-->>C: 支付成功
    C-->>S: 订单创建成功
```

### 3. 用户注册序列图

```mermaid
sequenceDiagram
    participant U as 用户
    participant W as Web前端
    participant V as 验证服务
    participant M as 邮件/短信服务
    participant A as 账户服务
    participant DB as 数据库

    U->>W: 填写注册信息
    W->>V: 发送验证码请求
    V->>M: 发送验证码
    M-->>U: 接收验证码
    U->>W: 输入验证码
    W->>V: 验证验证码
    V-->>W: 验证通过

    W->>A: 提交注册信息
    A->>DB: 检查用户名是否存在
    DB-->>A: 不存在
    A->>DB: 创建用户记录
    DB-->>A: 用户创建成功
    A->>A: 生成激活链接
    A->>M: 发送激活邮件
    M-->>U: 激活邮件

    U->>W: 点击激活链接
    W->>A: 激活请求
    A->>DB: 更新用户状态(已激活)
    DB-->>A: 更新成功
    A-->>W: 激活成功
    W-->>U: 注册完成
```

### 4. 并发处理序列图

```mermaid
sequenceDiagram
    participant C as 客户端
    participant L as 负载均衡器
    participant S1 as 服务实例1
    participant S2 as 服务实例2
    participant Q as 消息队列
    participant W as 工作者

    C->>L: HTTP请求
    L->>S1: 转发请求1
    L->>S2: 转发请求2

    par 并行处理
        S1->>S1: 处理业务逻辑
        S2->>S2: 处理业务逻辑
    end

    S1-->>L: 响应1
    S2-->>L: 响应2
    L-->>C: 聚合响应

    Note over L,C: 如果某个服务响应慢\n则使用快速返回的响应

    C->>L: 提交异步任务
    L->>Q: 发送到消息队列
    L-->>C: 任务已接收
    Q->>W: 消费消息
    W->>W: 处理耗时任务
    W-->>C: 通知任务完成(可选)
```

### 5. 错误处理序列图

```mermaid
sequenceDiagram
    participant C as 客户端
    participant A as API网关
    participant S as 业务服务
    participant D as 数据服务
    participant R as 降级服务

    C->>A: 发起请求
    A->>S: 转发请求
    S->>D: 查询数据

    alt 数据服务正常
        D-->>S: 返回数据
        S-->>A: 处理结果
        A-->>C: 正常响应
    else 数据服务超时
        D--xS: 超时
        S->>S: 尝试重试(1)
        S->>D: 重试请求
        D--xS: 再次超时
        S->>S: 尝试重试(2)
        S->>D: 重试请求
        D--xS: 第三次超时
    else 数据服务故障
        D--xS: 连接拒绝
    end

    alt 重试失败后的处理
        S->>R: 请求降级数据
        R-->>S: 返回缓存/默认值
        S-->>A: 降级响应
        A-->>C: 返回降级数据+友好提示
        Note over C: 展示部分数据\n提示稍后再试
    else 降级服务不可用
        S-->>A: 返回错误
        A-->>C: 500错误+错误码
        Note over C: 提示服务暂时不可用
    end
```

### 6. 微服务调用序列图

```mermaid
sequenceDiagram
    participant C as 客户
    participant G as API网关
    participant U as 用户服务
    participant O as 订单服务
    participant P as 商品服务
    participant I as 库存服务
    participant M as 消息队列

    C->>G: 下单请求
    G->>U: 验证用户(token)
    U-->>G: 用户有效

    G->>O: 创建订单
    O->>P: 查询商品信息
    P-->>O: 返回商品详情
    O->>I: 查询库存
    I-->>O: 库存充足

    O->>O: 创建订单记录
    O->>I: 锁定库存
    I-->>O: 锁定成功

    O->>M: 发送订单创建事件
    M-->>O: 事件已发送

    O-->>G: 订单创建成功
    G-->>C: 返回订单信息

    Note over M: 异步处理\n积分增加\n库存实际扣减\n发送通知邮件
```

## 使用指南

1. **确定参与者**：识别交互过程中的主要对象/服务
2. **确定消息**：识别参与者之间传递的信息
3. **确定顺序**：按时间顺序排列消息
4. **标注激活**：需要标识对象生命周期的开始和结束
5. **处理分支**：使用 `alt/else` 表示条件分支
6. **处理循环**：使用 `loop` 表示重复操作

## 最佳实践

- 消息流从上到下表示时间推进
- 使用清晰的参与者命名
- 关键消息添加说明注释
- 复杂流程拆分为多个序列图
- 使用 `Note` 标记重要说明
