# 活动图模板 (Activity Diagram)

## 模板说明

活动图（Activity Diagram）用于描述业务流程、工作流或算法的执行步骤，支持并行活动。

## 基本语法

```mermaid
graph TD
    A[开始] --> B[活动节点]
    B --> C{判断节点}
    C -->|条件1| D[活动1]
    C -->|条件2| E[活动2]
    D --> F[结束]
    E --> F

    subgraph 子图名称
        G --> H
    end
```

## 节点类型

| 语法 | 节点类型 | 说明 |
|------|----------|------|
| `[文本]` | 活动 | 圆角矩形，表示动作/操作 |
| `{文本}` | 判断 | 菱形，表示条件判断 |
| `([[文本]])` | 开始/结束 | 圆圈，表示流程开始或结束 |
| `((文本))` | 圆形 | 大圆点，表示流程开始(竖向) |
| `([文本])` | 跑道形 | 圆角矩形两端半圆，表示子流程 |

## 分支与循环

```mermaid
graph TD
    A --> B{条件判断}
    B -->|是| C[执行活动]
    B -->|否| D[其他活动]
    C --> D

    subgraph 循环示例
        E --> F{继续循环?}
        F -->|是| G[循环体]
        G --> E
        F -->|否| H[退出循环]
    end
```

## 模板示例

### 1. 用户注册流程

```mermaid
graph TD
    A([开始注册]) --> B[填写注册信息]
    B --> C{信息验证}
    C -->|失败| D[提示错误信息]
    D --> B
    C -->|成功| E[发送验证码]
    E --> F[输入验证码]
    F --> G{验证码正确?}
    G -->|否| H[提示验证码错误]
    H --> F
    G -->|是| I[创建账户]
    I --> J[发送欢迎邮件]
    J --> K([注册成功])
```

### 2. 订单处理流程

```mermaid
graph TD
    A([新订单]) --> B[验证订单信息]
    B --> C{信息有效?}
    C -->|否| D[通知用户修正]
    D --> B
    C -->|是| E[检查库存]

    E --> F{库存充足?}
    F -->|否| G[预留库存/等待补货]
    G --> H{库存满足?}
    H -->|是| E
    H -->|超时| I[取消订单]
    I --> J([订单取消])
    F -->|是| K[锁定库存]

    K --> L[计算费用]
    L --> M[发起支付]

    M --> N{支付成功?}
    N -->|否| O[订单待支付]
    O --> P[等待用户支付]
    P --> M
    N -->|是| Q[确认订单]

    Q --> R[发送订单确认通知]
    R --> S[安排发货]
    S --> T([订单完成])
```

### 3. 并行活动处理

```mermaid
graph TD
    A([开始]) --> B[接收订单]

    subgraph 并行处理
        B --> C[查询库存]
        B --> D[计算价格]
        B --> E[准备包装材料]
    end

    C --> F{库存足够?}
    D --> G[费用计算完成]
    E --> H[包装材料就绪]

    F -->|否| I[通知客户缺货]
    I --> J([处理结束])
    F -->|是| K[继续处理]

    G --> K
    H --> K
    K --> L[打包商品]
    L --> M[安排发货]
    M --> N([发货完成])
```

### 4. 审批流程（带角色）

```mermaid
graph TD
    A([提交申请]) --> B[提交给部门经理]

    subgraph 部门经理审批
        B --> C{经理审批通过?}
        C -->|否| D[返回修改意见]
        D --> E[修改申请]
        E --> B
        C -->|是| F[提交给总监]
    end

    subgraph 总监审批
        F --> G{总监审批通过?}
        G -->|否| H[返回修改意见]
        H --> E
        G -->|是| I[提交给财务]
    end

    subgraph 财务审批
        I --> J{财务审批通过?}
        J -->|否| K[返回修改意见]
        K --> E
        J -->|是| L[进行付款处理]
    end

    L --> M[更新申请状态]
    M --> N[通知申请人]
    N --> O([审批完成])
```

### 5. 会员等级计算

```mermaid
graph TD
    A([开始]) --> B[统计年度消费金额]
    B --> C[统计年度订单数量]
    C --> D[统计积分]

    D --> E{消费金额 >= 10000?}
    E -->|是| F[晋升为黄金会员]
    E -->|否| G{消费金额 >= 5000?}

    G -->|是| H[维持白银会员]
    G -->|否| I[维持普通会员]

    F --> J[应用黄金会员权益]
    H --> K[应用白银会员权益]
    I --> L[应用普通会员权益]

    J --> M[发送升级通知]
    K --> M
    L --> M

    M --> N([计算完成])
```

### 6. 登录与权限验证

```mermaid
graph TD
    A([用户访问]) --> B[显示登录页面]
    B --> C[输入用户名密码]

    C --> D[提交登录请求]

    D --> E{账户存在?}
    E -->|否| F[提示账户不存在]
    F --> B
    E -->|是| G[验证密码]

    G --> H{密码正确?}
    H -->|否| I[提示密码错误]
    I --> J[错误计数+1]
    J --> K{连续错误>=5?}
    K -->|是| L[锁定账户]
    L --> M[提示账户已锁定]
    M --> N([登录失败])
    K -->|否| B
    H -->|是| O[验证成功]

    O --> P{是否需要双因素?}
    P -->|是| Q[发送验证码]
    Q --> R[用户输入验证码]
    R --> S{验证码正确?}
    S -->|否| T[提示验证码错误]
    T --> Q
    S -->|是| U[验证通过]
    P -->|否| U

    U --> V[生成Token]
    V --> W[记录登录日志]
    W --> X([登录成功])
```

## 使用指南

1. **明确开始和结束**：每个流程有且只有一个开始，可有多个结束
2. **活动命名**：使用动宾短语（如"提交订单"、"验证信息"）
3. **判断节点**：用菱形表示，标注清晰的分支条件
4. **并行处理**：使用 `subgraph` 分组并行活动
5. **泳道活动图**：如需表示不同角色的活动，可用泳道（Swimlanes）

## Mermaid活动图限制

Mermaid 的 `graph TD` 语法支持：
- 开始/结束节点：`([文本])` 或 `[[文本]]`
- 活动节点：`[文本]`
- 判断节点：`{文本}`
- 子图：`subgraph`

不支持真正的并行活动图（Fork/Join），但可通过视觉分组表示并行概念。
