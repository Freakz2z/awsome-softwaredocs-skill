# 状态图模板 (State Diagram)

## 模板说明

状态图（State Diagram）用于描述对象或系统的状态变化，以及触发状态转换的事件。

## 基本语法

```mermaid
stateDiagram
    [*] --> State1: 初始转换
    State1 --> State2: 事件
    State2 --> [*]: 结束转换

    state State3 {
        [*] --> SubState1
        SubState1 --> SubState2: 事件
        SubState2 --> [*]: 完成
    }
```

## 状态图元素

| 语法 | 元素类型 | 说明 |
|------|----------|------|
| `[*]` | 初始状态 | 黑点，表示状态机开始 |
| `[*] --> State` | 转换箭头 | 带事件的转换 |
| `State1 --> State2: 事件` | 状态转换 | 触发条件和结果 |
| `state Name { }` | 复合状态 | 包含子状态的状态 |

## 模板示例

### 1. 订单状态图

```mermaid
stateDiagram-v2
    [*] --> 待支付

    待支付 --> 已支付: 支付成功
    已支付 --> 已取消: 用户取消/超时
    已支付 --> 已发货: 商家发货

    已发货 --> 运输中: 物流揽收
    运输中 --> 已到达: 到达目的地
    已到达 --> 派送中: 开始派送

    派送中 --> 已签收: 客户签收
    已签收 --> 已完成: 完成评价
    已签收 --> 退换货中: 申请退换货

    退换货中 --> 已退款: 审核通过
    退换货中 --> 已退货: 商家收到退货
    已退货 --> 已退款: 退款完成

    已取消 --> [*]
    已退款 --> [*]
    已完成 --> [*]
```

### 2. 用户账户状态图

```mermaid
stateDiagram-v2
    [*] --> 激活中

    激活中 --> 正常: 邮箱验证
    激活中 --> 失效: 验证超时

    状态 --> 正常: 管理员激活

    待审核 --> 正常: 审核通过
    待审核 --> 异常: 审核拒绝

    冻结 --> 正常: 解冻申请
    冻结 --> 封禁: 严重违规

    正常 --> 冻结: 触发风控
    正常 --> 待审核: 修改关键信息

    封禁 --> [*]
    失效 --> [*]
```

### 3. 线程状态图

```mermaid
stateDiagram-v2
    [*] --> 新建: new Thread()

    新建 --> 就绪: start()
    就绪 --> 运行: 获得CPU

    运行 --> 就绪: 时间片用完
    运行 --> 阻塞: wait()/sleep()/IO
    运行 --> 等待锁: synchronized

    阻塞 --> 就绪: 条件满足
    等待锁 --> 就绪: 获得锁

    运行 --> 终止: run()结束
    等待锁 --> 终止: uncaughtException
```

### 4. 复合状态图（订单生命周期）

```mermaid
stateDiagram-v2
    [*] --> 待支付

    state 待支付 {
        [*] --> 初始化
        初始化 --> 计算中: 计算费用
        计算中 --> 等待支付: 生成订单
        等待支付 --> 超时: 30分钟未支付
        初始化 --> 取消: 用户取消
        超时 --> [*]: 自动取消
        取消 --> [*]
    }

    待支付 --> 已支付: 支付成功

    state 已支付 {
        [*] --> 支付确认
        支付确认 --> 准备中: 确认支付
        准备中 --> 等待发货: 商品准备完毕
    }

    已支付 --> 交易关闭: 退款取消

    已支付 --> 已发货: 商家发货

    state 已发货 {
        [*] --> 仓库发货
        仓库发货 --> 运输中: 物流揽收
        运输中 --> 派送中: 到达目的站
        派送中 --> 等待签收: 快递员配送
    }

    已发货 --> 退换货: 申请售后
    派送中 --> 已签收: 客户签收

    state 退换货 {
        [*] --> 申请中
        申请中 --> 审核中: 提交申请
        审核中 --> 通过: 审核通过
        审核中 --> 拒绝: 审核不通过
        通过 --> 退货中: 客户退货
        拒绝 --> [*]
        退货中 --> 收到退货: 商家收货
        收到退货 --> 退款中: 验货完成
        退款中 --> 已退款: 退款完成
    }

    已签收 --> 已完成: 完成交易
    已签收 --> 退换货: 申请售后
    已完成 --> [*]
    已退款 --> [*]
```

### 5. 文档状态图（工作流）

```mermaid
stateDiagram-v2
    [*] --> 草稿

    草稿 --> 审核中: 提交审核
    审核中 --> 审核通过: 审核人批准

    发布中 --> 已发布: 执行发布

    审核中 --> 审核拒绝: 审核人驳回

    草稿 --> 废弃: 作者废弃
    审核拒绝 --> 废弃: 作者确认废弃

    已发布 --> 废弃: 紧急下架
    已发布 --> 更新中: 发起修订
    更新中 --> 已发布: 重新发布

    废弃 --> [*]

```

### 6. HTTP请求状态图

```mermaid
stateDiagram-v2
    [*] --> 空闲

    空闲 --> 请求中: 发起请求

    请求中 --> 成功: 2xx响应
    请求中 --> 客户端错误: 4xx响应
    请求中 --> 服务器错误: 5xx响应
    请求中 --> 网络错误: 连接失败/超时
    请求中 --> 取消: abort()

    客户端错误 --> 重试中: 自动重试
    客户端错误 --> 请求结束: 手动处理

    服务器错误 --> 重试中: 自动重试
    服务器错误 --> 请求结束: 超过重试次数

    重试中 --> 请求中: 重试计数<3
    重试中 --> 请求结束: 重试次数>=3

    网络错误 --> 重试中: 自动重试
    网络错误 --> 请求结束: 超过重试次数

    成功 --> 请求结束: 处理响应
    取消 --> 请求结束

    请求结束 --> [*]
```

## 使用指南

1. **识别状态**：确定对象可能处于的所有状态
2. **识别转换**：确定状态之间的转换条件和事件
3. **初始和最终状态**：明确状态的起点和终点
4. **复合状态**：对于复杂对象，使用嵌套的子状态
5. **动作标注**：可在转换箭头旁标注动作（guard conditions）

## 状态图与活动图的区别

| 特征 | 状态图 | 活动图 |
|------|--------|--------|
| 关注点 | 对象的状态变化 | 活动的执行流程 |
| 节点 | 状态 | 活动 |
| 转换 | 事件触发 | 按顺序执行 |
| 并行 | 一般不支持 | 支持并行活动 |
| 适用场景 | 状态机、生命周期 | 业务流程、工作流 |
