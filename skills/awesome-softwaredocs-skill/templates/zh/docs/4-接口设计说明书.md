## 2. REST API 设计

### 2.1 API 规范

#### 2.1.1 基本规范

- **基础URL**：`https://api.example.com/v1`
- **认证方式**：Bearer Token (JWT)
- **请求格式**：JSON (`Content-Type: application/json`)
- **响应格式**：JSON
- **字符编码**：UTF-8

#### 2.1.2 通用响应格式

```json
{
    "code": 200,
    "message": "success",
    "data": {},
    "timestamp": "2024-01-01T12:00:00Z",
    "requestId": "uuid"
}
```

#### 2.1.3 错误响应格式

```json
{
    "code": 400,
    "message": "参数错误",
    "errors": [
        {
            "field": "username",
            "message": "用户名不能为空"
        }
    ],
    "timestamp": "2024-01-01T12:00:00Z",
    "requestId": "uuid"
}
```

#### 2.1.4 HTTP 状态码

| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 201 | 创建成功 |
| 400 | 请求参数错误 |
| 401 | 未认证 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 2.2 用户管理模块

#### 2.2.1 用户相关接口

##### 获取用户列表

```
GET /users
```

**请求参数**：
| 参数名 | 类型 | 位置 | 必填 | 说明 |
|--------|------|------|------|------|
| page | int | query | 否 | 页码，默认1 |
| pageSize | int | query | 否 | 每页条数，默认20 |
| keyword | string | query | 否 | 搜索关键词 |

**响应示例**：
```json
{
    "code": 200,
    "data": {
        "list": [
            {
                "id": "uuid",
                "username": "zhangsan",
                "email": "zhangsan@example.com",
                "status": "ACTIVE",
                "createdAt": "2024-01-01T12:00:00Z"
            }
        ],
        "pagination": {
            "page": 1,
            "pageSize": 20,
            "total": 100,
            "totalPages": 5
        }
    }
}
```

##### 获取用户详情

```
GET /users/{id}
```

**路径参数**：
| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | string | 用户ID |

**响应示例**：
```json
{
    "code": 200,
    "data": {
        "id": "uuid",
        "username": "zhangsan",
        "email": "zhangsan@example.com",
        "phone": "13800138000",
        "status": "ACTIVE",
        "roles": ["ADMIN", "USER"],
        "createdAt": "2024-01-01T12:00:00Z",
        "updatedAt": "2024-01-01T12:00:00Z"
    }
}
```

##### 创建用户

```
POST /users
```

**请求体**：
```json
{
    "username": "zhangsan",
    "password": "Password123!",
    "email": "zhangsan@example.com",
    "phone": "13800138000",
    "roleIds": ["role-1", "role-2"]
}
```

**响应示例**：
```json
{
    "code": 201,
    "data": {
        "id": "uuid",
        "username": "zhangsan"
    }
}
```

##### 更新用户

```
PUT /users/{id}
```

##### 删除用户

```
DELETE /users/{id}
```

## 6. 接口安全

### 6.1 认证机制

| 接口类型 | 认证方式 | 说明 |
|----------|----------|------|
| 外部API | JWT Token | Bearer Token |
| 内部API | Service Auth | 服务间认证 |
| 第三方API | OAuth2/API Key | 按服务商 |

### 6.2 限流策略

| 限流维度 | 限制值 | 时间窗口 |
|----------|--------|----------|
| IP维度 | 1000次 | 1分钟 |
| 用户维度 | 100次 | 1分钟 |
| 接口维度 | [值] | [窗口] |

### 6.3 敏感数据处理

- 密码：加密存储，接口不返回
- 手机号：脱敏处理（138****8000）
- 身份证：脱敏处理
- 银行卡：脱敏处理
